What if an attacker tries this?

Instead of a real transaction ID, the attacker enters:

' OR '1'='1

This forces the system to return ALL transactions instead of just one.

✅ Secure Solution

Instead of concatenating user input into a query like this:

query = "SELECT * FROM transactions WHERE id = '" + userInput + "'";

We should use **Prepared Statements**:

query = "SELECT * FROM transactions WHERE id = ?";

🔐 This prevents attackers from injecting malicious SQL!
